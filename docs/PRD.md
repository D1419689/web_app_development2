# PRD - 個人記帳簿

## 1. 專案概述

### 背景與動機
在快節奏的現代生活中，許多人往往無法有效追蹤自己的日常開銷，導致月底時常有「錢都花去哪了」的疑惑。為了幫助使用者建立良好的理財習慣，我們需要一個簡單、直覺的收支管理系統，讓記錄花費變得輕鬆無負擔。

### 目標用戶
想要掌握自己財務狀況、進行預算控管的學生或上班族。

### 核心價值主張
這是一個專為個人設計的收支管理系統，旨在透過簡單的介面，協助使用者快速記錄每日開銷與收入，並透過圖像化報表分析財務狀況，進而達到有效控管預算的目的。

---

## 2. 功能需求

以下為系統的主要功能與對應的使用者故事：

### 1. 每日收支記錄
作為一名 **使用者**，我希望 **能夠快速輸入金額、類別（食、衣、住、行等）與日期**，以便 **記錄我每天的日常收支**。

### 2. 圖像化報表分析
作為一名 **使用者**，我希望 **系統能自動生成圓餅圖或柱狀圖**，以便 **直觀地分析我每月的支出比例與財務狀況**。

### 3. 預算設定與超支提醒
作為一名 **使用者**，我希望 **能夠設定每月總預算**，以便 **當消費接近限額時能收到警告通知，避免超支**。

### 4. 歷史紀錄搜尋與匯出
作為一名 **使用者**，我希望 **可依日期或類別篩選過往資料，並支援匯出成 CSV 檔案**，以便 **進行更深入的財務回顧與資料備份**。

### 5. 多帳戶管理
作為一名 **使用者**，我希望 **系統支援區分現金、信用卡及銀行存款等多種資金來源**，以便 **更精確地管理不同帳戶的餘額與流向**。

---

## 3. 非功能需求

### 技術限制
- 後端框架：Flask (Python)
- 模板引擎：Jinja2
- 資料庫：SQLite

### 效能與安全考量
- 系統回應時間應在合理範圍內，確保使用者能流暢地記錄收支。
- 確保使用者輸入的資料安全，防範基本的網頁攻擊（如 SQL Injection, XSS 等）。
- 匯出的 CSV 檔案格式應正確且相容於常見的試算表軟體（如 Excel, Google Sheets）。

---

## 4. MVP 範圍 (Minimum Viable Product)

### Must Have (必要功能)
- 每日收支記錄（新增、修改、刪除）。
- 圖像化報表分析（至少提供一種圖表呈現當月支出比例）。

### Should Have (應有功能)
- 預算設定與超支提醒（基本的預算設定與簡單的警示提示）。
- 歷史紀錄搜尋與列表顯示。

### Nice to Have (加分功能)
- 歷史紀錄匯出成 CSV 檔案。
- 多帳戶管理（現金、信用卡等分類）。
